# Прикладной Python: Проект №3

## Описание проекта

В проекте реализован FastAPI-сервис для сокращения URL-адресов. Сервис позволяет пользователям создавать короткие ссылки, которые перенаправляют на длинные URL-адреса, искать уже созданные короткие ссылки и смотреть по ним статистику. Для зарегистрированных пользователей доступна возможность просматривать историю своих ссылок, а также удалять и изменять их.

---

## Структура проекта

Все данные по ссылкам и пользователям хранятся в базе данных PostgreSQL. Для миграций используется Alembic. Кэширование популярных ссылок реализовано с помощью Redis, что позволяет ускорить перенаправление по коротким URL-адресам.

Для удаления просроченных ссылок используется Celery, который периодически проверяет базу данных и удаляет устаревшие записи. Также Celery используется для обновления кэша, производя периодическую проверку популярности ссылок.

## Модели базы данных

### `user` — Зарегистрированные пользователи

Таблица создаётся библиотекой `fastapi-users` с полями электронной почты, хэша пароля и флагов активности/верификации. Добавлено дополнительное поле:

| Поле | Тип | Описание |
|---|---|---|
| `id` | UUID | Первичный ключ |
| `email` | string | Адрес электронной почты (уникальный) |
| `hashed_password` | string | Хэш пароля |
| `is_active` | bool | Активен ли аккаунт |
| `is_verified` | bool | Подтверждён ли email (не используется) |
| `is_superuser` | bool | Суперпользователь (не используется) |
| `registered_at` | timestamptz | Дата и время регистрации |

### `short_links` — Активные короткие ссылки

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | Первичный ключ |
| `original_url` | string | Оригинальный URL |
| `short_code` | string | Уникальный короткий код (псевдоним) |
| `created_at` | timestamptz | Дата создания |
| `last_accessed_at` | timestamptz | Дата последнего перехода |
| `access_count` | integer | Количество переходов |
| `expires_at` | timestamptz | Дата истечения ссылки |
| `owner_id` | UUID (FK) | Владелец ссылки (NULL для анонимных) |

### `expired_links` — История ссылок

Хранит архив ссылок, удалённых вручную владельцем или по истечении срока действия.

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | Первичный ключ |
| `original_url` | string | Оригинальный URL |
| `short_code` | string | Короткий код |
| `created_at` | timestamptz | Дата создания ссылки |
| `expired_at` | timestamptz | Дата удаления/истечения |
| `access_count` | integer | Количество переходов за время жизни |
| `deleted_by_user` | bool | `true` — удалена вручную, `false` — истёк срок |
| `owner_id` | UUID (FK) | Владелец ссылки |

## Эндпоинты авторизации

### `POST /auth/register` — Регистрация нового пользователя

Валидация полей:
- `email` — должен содержать символ `@`
- `password` — длина от 8 до 128 символов

```json
{
  "email": "user@example.com",
  "password": "strongpassword"
}
```

### `POST /auth/jwt/login` — Вход в систему, возвращает JWT-токен

```
Content-Type: application/x-www-form-urlencoded

username=user@example.com&password=strongpassword
```

### `POST /auth/jwt/logout` — Выход из системы (требует авторизации)

```
Authorization: Bearer <token>
```

## Эндпоинты для работы с короткими ссылками

### `POST /links/shorten` — Создание короткой ссылки

Доступно для всех пользователей. Авторизованные пользователи становятся владельцами ссылки и могут её изменять и удалять. Поля `expires_at` и `custom_alias` — необязательные.

Если `expires_at` не указан, ссылка истекает через 30 дней с момента создания. Поддерживаемые форматы:

| Формат | Пример |
|---|---|
| `YYYY-MM-DD HH:MM` | `2026-12-31 23:59` |
| `YYYY-MM-DDTHH:MM:SSZ` | `2026-12-31T23:59:00Z` |
| `YYYY-MM-DDTHH:MM:SS` | `2026-12-31T23:59:00` |

```json
{
  "url": "https://example.com/very/long/url",
  "expires_at": "2026-12-31 23:59",
  "custom_alias": "my-link"
}
```

### `GET /links/search?original_url=<url>` — Поиск коротких ссылок по оригинальному URL

```
GET /links/search?original_url=https://example.com/very/long/url
```

### `GET /links/{short_code}` — Перенаправление по короткой ссылке

```
GET /links/my-link
```

Возвращает редирект `302` на оригинальный URL. Популярные ссылки кэшируются в Redis.

### `GET /links/{short_code}/stats` — Статистика по короткой ссылке

```
GET /links/my-link/stats
```

Возвращает информацию о ссылке: оригинальный URL, дату создания, дату истечения, дату последнего обращения и количество переходов.

### `PUT /links/{short_code}?new_url=<url>` — Обновление оригинального URL (требует авторизации, только для владельца)

```
PUT /links/my-link?new_url=https://new-example.com
Authorization: Bearer <token>
```

### `DELETE /links/{short_code}` — Удаление ссылки (требует авторизации, только для владельца)

Удалённая ссылка сохраняется в историю.

```
DELETE /links/my-link
Authorization: Bearer <token>
```

### `POST /links/history` — История ссылок пользователя (требует авторизации)

Возвращает список истекших и удалённых ссылок пользователя с пагинацией.

```json
{
  "page": 1,
  "limit": 20
}
```

```
Authorization: Bearer <token>
```

## Запуск приложения

### Требования

- Docker и Docker Compose

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd AppliedPython_Project_3
```

### 2. Создать файл `.env`

Создайте файл `.env` в корне проекта со следующим содержимым:

```dotenv
DB_USER=postgres
DB_PASS=postgres
DB_HOST=db_app
DB_PORT=1221
DB_NAME=url_shortener

AUTH_SECRET=your-secret-key

REDIS_URL=redis://redis_app:5370
```

### 3. Запустить контейнеры

```bash
docker compose up --build
```

После запуска:

| Сервис | Адрес |
|---|---|
| API (FastAPI) | http://localhost:9999 |
| Документация Swagger | http://localhost:9999/docs |
| Flower (мониторинг Celery) | http://localhost:5555 |

### 4. Применить миграции

Миграции применяются автоматически при старте контейнера `app` (скрипт `docker/app.sh` выполняет `alembic upgrade head` перед запуском сервера).


### Развертывание проекта

Проект развернут на сервере: `http://5.35.95.72:9999`
